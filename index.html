01About

A builder with an unusual edge

Most of what I know, I learned because I needed it. A pile of clips needed naming, so I built a tool that names them. Hours of sermon footage needed cutting, so I built something that finds the good parts. That's the pattern — hit a real problem, build the thing, ship it.

Under the hood that's meant computer vision, speech-to-text, LLMs, and full-stack web work — Python and TypeScript, FastAPI and React, Docker, the usual suspects. But I care less about the stack than whether the thing actually works for the person using it.
